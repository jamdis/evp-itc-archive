Hi Steve,

the CW decoder gets mainly the letters E, I and T.
I also noticed that the beeps are all the same length in your file.

So I made a raw audio file for EVPmaker that contains three cues: The
"dit" and "dah" beeps, and an "intra-character gap":

http://groups.yahoo.com/group/EVP-ITC/files/Stefan/Morse%20code/morse.wav

The "Use Cue List" option must be checked in EVPmaker to use this file.

Now it sounds a bit more like actual morse code, and CwGet is able to
decode more letters. Perhaps some former ham operators "over there" are
able to use it... :)
